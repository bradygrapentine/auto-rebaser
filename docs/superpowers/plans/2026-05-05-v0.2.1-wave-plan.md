# v0.2.1 Wave Plan — Phase 5 remainder
_2026-05-05_

Ships stories **5.4**, **5.5**, **5.1** as a single release on top of `main` (which already has 5.6 activity log).

## Execution mode: direct, serial

Three tracks, but **AutomationsSettings.tsx**, **PRRow.tsx**, and **settings-store.ts** are touched by 2–3 of them. Parallel dispatch = guaranteed merge churn. Per global CLAUDE.md ("default to direct implementation… only dispatch for 4+ truly independent parallel tracks"), do these serially in one Opus session.

## Order: 5.4 → 5.5 → 5.1

Rationale:
- **5.4 first** because it's a settings-shape change (`autoMergeMethod` → `mergeMethodPreference[]`) with a one-shot migration. Land the migration before adding more settings UI (5.1, 5.5) on top.
- **5.5 second** — pure-add popup hook, no schema impact, smallest blast radius.
- **5.1 last** — biggest: new endpoint, new full-popup view, throttle store, settings additions. Benefits from the keyboard-shortcut wiring already in place.

## Risks
- **5.1 throttle correctness** — easy to mis-key on PR id vs node id. Test explicitly with two PRs sharing a number across repos.
- **5.4 migration** — first user-load after upgrade must not lose their existing `autoMergeMethod` value. Migration test is the gate.
- **5.5 focus stealing** — keydown handler must not intercept while the user types in settings textareas (5.1 ping template). The `editable target` skip rule covers it; verify with a test.
